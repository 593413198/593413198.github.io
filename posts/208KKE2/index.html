概要

什么是Compiler(编译器)与Interpreter(解释器)
两者的区别与优缺点，并以常用的（高级）语言为例

什么是Compiler？

  • Compilers are programs that convert high-level language(source code) to machine language code before executing it.

提取一下关键词：

    1. programs
      编译器也是一个程序，也是又某一种Language实现的；
      所以不要将其看得过于抽象！
    1. high lanage -> machine language
      将高级语言转化为可执行的机器语言，当然中间省略了一些步骤；
      可查阅： The Four Stages of Compiling a C program
    1. before executing it
      这是区别于解释器最大的地方，先全部编译成机器语言，以后要用了随时执行！

什么是Interpreter？

  • Interpreters are also programs that convert high-level language(source code) to machine language code, but compile and execute for each line.

提取一下关键词：

  • 1.2点都与Compiler一样，即从高级语言转化到机器语言;
    1. compile and execute for each line
      解释器不会产生任何中间代码，一旦编译一行，便开始立即执行；
      你应该发现，python的报错总是出现只有一处，而C++却能同时报好多的warning和erro

Self-Reference

什么是自举？即一个语言能自己解释自己；
举个例子：用C语言实现了python，再用python写一个python解释器，那python的编译运行就可以完全脱离C了（这就是pypy)

Compiler和Interpreter的区别

左对齐 Compiler Interpreter
步骤 先编译后, (再执行) 编译与执行一起
特点 better performance memory efficient
运行速度 快 慢 (常见语言的性能对比)
硬盘 大：存储机器码 小：不保存任何机器码 (.pyc)
内存 大：生成中间代码 小
error report 所有报错一起出现 单行报错即阻碍 (python try)
常见语言 C, C++, C#, Java, Go python, js, lua
